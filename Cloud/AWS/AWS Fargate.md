# AWS Fargate

AWS Fargate는 컨테이너를 위한 서버리스 컴퓨팅 엔진이다. EC2 인스턴스를 직접 프로비저닝하거나 관리하지 않고 컨테이너 자체에만 집중할 수 있게 해준다. [[Amazon ECS]] 또는 [[Amazon EKS]]와 함께 동작하며, 서버가 없는 것이 아니라 AWS가 서버를 대신 관리해준다는 의미에서 "서버리스"라고 부른다.

## 등장 배경

Docker 컨테이너를 AWS에서 실행하려면 어딘가의 서버에서 돌아야 한다. Fargate 이전의 방식인 ECS on EC2에서는 컨테이너를 실행하기 위해 EC2 인스턴스 프로비저닝, 오토스케일링 그룹 설정, 보안 패치, [[Amazon EC2#Bin Packing|bin packing]] 계산 등을 직접 관리해야 했다. Fargate는 이 모든 서버 관리 부담을 없앤다.

## 핵심 개념

Fargate에서 컨테이너를 실행하는 단위는 Task Definition, Task, Service 세 가지로 구성된다.

**Task Definition(태스크 정의)** 은 컨테이너를 어떻게 실행할지 기술하는 청사진이다. 어떤 Docker 이미지를 사용할지, vCPU와 메모리를 얼마나 할당할지, 환경변수는 무엇인지 등을 정의한다.

**Task(태스크)** 는 Task Definition의 실행 인스턴스다. 실제로 실행 중인 컨테이너(들)의 묶음이며, Task Definition을 기반으로 생성된다.

**Service(서비스)** 는 태스크를 항상 N개 유지하도록 관리하는 단위다. 태스크가 종료되면 자동으로 새로운 태스크를 시작해 지정한 수를 유지한다.

## EC2 방식과의 운영 차이

Fargate를 사용하면 서버 관리 부담은 사라지지만, EC2 방식과 다른 운영 방식을 새롭게 익혀야 한다.

**디버깅** 방식이 달라진다. EC2에서는 SSH로 인스턴스에 직접 접속하여 로그를 확인하고 프로세스를 점검할 수 있다. Fargate는 접속할 서버 자체가 없기 때문에 CloudWatch Logs로 로그를 확인하거나, ECS Exec을 통해 실행 중인 컨테이너에 명령을 보내야 한다.

**스토리지** 모델도 다르다. EC2는 EBS 볼륨을 붙여 데이터를 영구 저장할 수 있다. Fargate 태스크는 기본적으로 태스크가 종료되면 로컬 스토리지도 함께 사라지므로, 영구 저장이 필요하면 EFS(Elastic File System)를 별도로 마운트해야 한다.

**네트워킹** 단위가 다르다. EC2 방식에서는 인스턴스 하나에 ENI(Elastic Network Interface)가 붙고 그 위의 컨테이너들이 네트워크를 공유한다. Fargate는 태스크마다 ENI가 하나씩 할당되므로, 태스크마다 고유한 IP와 보안 그룹을 가진다. 태스크 수가 많아지면 AWS 계정의 ENI 한도에 영향을 줄 수 있다.

**시작 시간(Cold Start)** 도 차이가 있다. EC2 방식에서는 인스턴스가 이미 실행 중이면 컨테이너가 수 초 안에 시작된다. Fargate는 태스크 실행마다 환경을 새로 구성하므로 30초에서 1분 이상 걸릴 수 있으며, Docker 이미지가 클수록 시작 시간이 길어진다.

## 과금 모델

EC2처럼 인스턴스 단위로 시간당 요금이 청구되는 것이 아니라, 태스크가 실제로 실행된 시간 동안 사용한 vCPU와 메모리 양에 따라 청구된다. 태스크가 실행되지 않으면 비용이 발생하지 않는다.
